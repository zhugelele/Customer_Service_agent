# 🤖 [autoAgent] - 基于 LLM 的自动化客服助手

本系统是一个基于大语言模型（LLM）驱动的自动化客服机器人，旨在实现平台客服的智能化值守与协同决策。**本项目核心架构参考并致谢 [XianyuAutoAgent](https://github.com/shaxiu/XianyuAutoAgent) 项目**，并在此基础上进行了针对个人需求的功能迭代与优化。

## 💡 项目起源

本项目旨在通过 LLM 技术实现高效率的自动化客户接待。参考了 `shaxiu` 等开发者的优秀开源实践，我们构建了一套支持多专家协同、上下文感知及智能议价的自动化处理流，以解决日常处理中的重复性劳动。

## 🌟 核心特性

| **功能模块**   | **实现逻辑**        | **亮点**                                               |
| -------------- | ------------------- | ------------------------------------------------------ |
| **上下文感知** | 会话历史管理        | 轻量级记忆模块，确保 LLM 的回复具备连贯的语境          |
| **专家路由**   | 意图识别 + 动态分发 | 通过 Prompt Engineering 区分议价、客服、技术咨询等场景 |
| **策略议价**   | 阶梯降价算法        | 预设价格底线与降价梯度，实现自动化议价                 |
| **辅助决策**   | 联网搜索整合        | 实时获取外部信息，增强回复的专业度和准确性             |

## 🚀 快速开始

### 环境依赖

- Python 3.8+

### 安装步骤

1. **克隆代码**

   Bash

   ```
   git clone [你的仓库地址]
   cd [你的项目文件夹名]
   ```

2. **安装依赖**

   Bash

   ```
   pip install -r requirements.txt
   ```

3. **环境配置**

   创建 `.env` 文件（或重命名 `.env.example`），配置必要参数：

   Code snippet

   ```
   API_KEY=你的API密钥
   COOKIES_STR=你的网页端Cookies
   MODEL_BASE_URL=模型API地址
   MODEL_NAME=模型名称
   # 可选：是否模拟人工输入延迟
   SIMULATE_HUMAN_TYPING=True
   ```

4. **自定义 Prompt**

   你可以根据实际业务场景，修改 `prompts/` 目录下的 `.txt` 文件，调整不同专家（分类、议价、技术等）的回复逻辑。
